Accessibility statement for Personal Independence Payment

This accessibility statement applies to the Personal Independence Payment service.

This service is run by the Department for Work and Pensions (DWP). We want as many people as possible to be able to use this service. For example, that means you should be able to:

  • change colours, contrast levels and fonts using browser or device settings
  • zoom in up to 400% without the text spilling off the screen
  • navigate most of the website using a keyboard or speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

We’ve also made the website text as simple as possible to understand.

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this website is

We know some parts of this website are not fully accessible:

  • some data inputs are not retained when a validation error occurs, or when a user navigates away from a page and then selects the ‘back’ link to return to it
  • sortable table headings and content are not fully accessible to screen reader software
  • it is not immediately clear to screen readers that a validation error is present on the screen
  • you can only skip to hyperlinks and sortable column headings on some of the pages

Feedback and contact information

We are always looking to improve the accessibility of this service. If you find any problems that are not listed on this page or think we're not meeting accessibility requirements email quarryhouse.applyforpip-accessibility@dwp.gov.uk.

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the 'accessibility regulations'). If you are not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about this website’s accessibility

The Department for Work and Pensions is committed to making its website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

This website is partially compliant with the Web Content Accessibility Guidelines version 2.2 AA standard, due to the non-compliances listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

Retaining data

Data that has been entered in one of the text fields is not retained in the text field when a validation error occurs, or when the user navigates to another page and then selects the ‘Back’ link to return to it. This fails WCAG 2.2 success criterion 3.3.7 (redundant entry).

Radio selection on one of the pages is not retained when a user navigates to another page and then selects the ‘Back’ link to return to it. This fails WCAG 2.2 success criterion 3.3.7 (redundant entry).

We plan to update these pages so that radio selections and text inputs are retained when validation errors occur, and when users navigate back to each page, by December 2024.

Validation errors

It is not immediately clear to screen readers that a validation error is present on the screen. On pages that show an error message, the page titles do not have an 'Error:' prefix. This fails WCAG 2.2 success criterion 3.3.1 (error identification).

We plan to add the ‘Error:’ prefix to the page titles of all pages with validation errors by December 2024.

Navigation

When using a screen reader’s alternative navigation with visual focus, the screen reader only skips to links and sortable column headings. It skips past all other page content. This fails WCAG 2.2 success criterion 2.4.3 (focus order).

When a table is sorted, the screenreader’s focus moves to the very top of the page, instead of back to the table. This fails WCAG 2.2 success criterion 2.4.3 (focus order).

We plan to change the table so that screen readers can consistently navigate through all of the content in the table without skipping, and so that the focus moves to the top of the table instead of the top of the page when the sort order changes, by December 2024.

Sortable table headings

Sortable headings do not have aria labels that are recognised by screen reader software. This fails WCAG 2.2 success criterion 1.3.1 (info and relationships), as well as WCAG 2.2 success criterion 4.1.2 (name, role, value).

Sortable headings do not state the column’s current sorting order. This fails WCAG 2.2 success criterion 1.3.1 (info and relationships), as well as WCAG 2.2 success criterion 4.1.2 (name, role, value).

The screen reader doesn’t describe what action will be performed when users select the sortable header, or how to interact with this feature. Instead, the voiceover only describes “down pointing black pointer”. This fails WCAG 2.2 success criterion 1.3.1 (info and relationships), as well as WCAG 2.2 success criterion 4.1.2 (name, role, value).

When the sorting order of a column is changed, there is no announcement for screen readers about the change in sorting order. This fails WCAG 2.2 success criterion 1.3.1 (info and relationships), as well as WCAG 2.2 success criterion 4.1.2 (name, role, value).

We plan to update sortable tables so that screen readers recognise each sortable heading’s aria label, state the current sorting order, describe how to interact with the sorting arrows, and announce any changes to the sorting order when these happen. We plan to make these changes by December 2024.

What we’re doing to improve accessibility

We conduct manual and automated accessibility testing for all major releases against the WCAG 2.2 criteria.

We are working to make improvements to the non-accessible content described in this statement by December 2024. We aim to make all pages retain data that has been entered previously, indicate which pages have validation errors, allow users to navigate through tables using visual focus, and format sortable tables so that the status and function of interactive elements are recognised and described by screen readers.

We will continue to iterate the design of the service to fix issues we are aware of.

Preparation of this accessibility statement

This statement was prepared on 30 September 2024. It was last reviewed on 30 September 2024.

The test was carried out by the Department for Work and Pensions. A sample of different types of pages were tested using automated and manual testing tools by our digital delivery teams.
